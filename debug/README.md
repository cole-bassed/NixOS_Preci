# Debug folder

This folder is for troubleshooting reports, evaluation notes, command captures, and repair plans for the dotfiles refactor.

Rules for this folder:

- Keep reports evidence-based: include exact commands, exit status, and the first concrete error.
- Do not mix implementation changes into troubleshooting reports.
- Prefer dated filenames: `YYYY-MM-DD-topic.md`.
- Use this folder for non-switching review/test reports while the repo is in flux.
